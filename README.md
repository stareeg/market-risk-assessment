# Проектная работа №2. Рыночный риск

Оценка рыночного риска портфеля из ОФЗ, акций и валюты. Пайплайн: сбор данных
(MOEX, ЦБ РФ), выделение риск-факторов через PCA, стохастические модели,
оценка стоимости инструментов, Monte Carlo для VaR и ES, бэктестинг и
статистические тесты. Курс НИУ ВШЭ.

Полное условие лежит в `project_tasks/task_description.md`.

## Как запустить

Поставить зависимости:

```bash
python -m pip install -r requirements.txt
```

Запустить весь пайплайн или отдельный этап:

```bash
python main.py --stage all       # все этапы по порядку
python main.py --stage data      # только сбор данных
python main.py --list            # список этапов и их статус
```

Этапы идут строго по порядку, потому что каждый читает parquet, сохранённый
предыдущим. Все параметры (даты, список инструментов, seed) лежат в
`config.py`, состав данных меняем только там.

Лучше работать в отдельном окружении: `python -m venv .venv`, активировать и
поставить зависимости. Папка `.venv` в .gitignore, в репозиторий не попадает.

## Структура репозитория

Пакеты этапов лежат прямо в корне, без обёртки src.

```
main.py            единая точка входа
requirements.txt   зависимости
refactor-plan.md   пошаговый план доработок и починки
config.py          параметры эксперимента
data_collection/   п.1 сбор данных
risk_factors/      п.2 риск-факторы и PCA
models/            п.3 стохастические модели
pricing/           п.4 оценка инструментов
var_engine/        п.5 Monte Carlo, VaR и ES
backtesting/       п.6 бэктестинг
stat_tests/        п.7 статистические тесты
viz/               графики для презентации
data/              parquet с промежуточными и финальными результатами
docs/              документация по данным, план команды, картинки
legacy/            старые тетрадки, источник для переноса, потом удалим
```

## Главные проблемы

Коротко, подробный разбор и план починки в `refactor-plan.md`.

- В п.3 две модели сразу: GARCH-DCC на PCA-факторах и отдельно CIR+GBM в обход
  PCA. Это противоречит п.2. Решили оставить единый подход GARCH-DCC на PCA.
- В п.4 цена облигаций считается по 3 точкам кривой, на длинных ОФЗ ошибка до
  1.5%. Кривую можно восстанавливать из PCA-нагрузок целиком.
- В п.4 нет кода, который создаёт часть вспомогательных parquet (коэффициенты
  факторной модели акций и последние цены). Это ломает воспроизводимость.
- В п.5 расчёт VaR нерабочий: дата берётся как «сегодня» вместо 2 декабря 2025,
  нет горизонта 1 день, нет ежедневной ребалансировки, DCC по сути отключён,
  переоценка идёт медленным циклом, уровни мер риска перепутаны (нужно VaR 99%
  и ES 97.5%).
