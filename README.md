## Проблемы и пробелы

### КРИТИЧНО: Две конфликтующие модели в пункте 3

В `risk_simulation.ipynb` **последовательно реализованы два разных подхода**, которые противоречат друг другу:

**Подход A** (первая половина тетрадки):
- GARCH(1,1)-t для всех 8 PCA-факторов → DCC для их корреляций
- Логически связан с пунктом 2 (работает с теми же PCA-scores)

**Подход B** (вторая половина тетрадки):
- CIR для 3 точек кривой доходности (0.25, 2, 10 лет) — работает с **уровнями ставок**, а не с PCA
- GBM для валют — простая модель без GARCH
- GARCH-DCC только для 3 факторов акций
- Общая корреляционная матрица инноваций 8×8

**Проблема:** Подход B **обходит PCA** для ставок и валюты, что противоречит пункту 2, где было обосновано использование PCA. Подход A логически целостнее, но пункт 5 (симуляция) написан на базе подхода B.

**Рекомендация:** Определиться с одним подходом. Подход A (GARCH-DCC-t на PCA-факторах) проще, целостнее и лучше обоснован. Если хочется CIR для ставок, нужно обосновать, почему PCA больше не используется.

### Пункт 3: Автокорреляция в остатках не решена

Диагностика Ljung-Box показала значимую автокорреляцию (p < 0.05) для:
- `RATE_PC1` (p=0.0003)
- `RATE_PC3` (p=0.037)
- `FX_EUR` (p=0.001)
- `FX_USD` (p=0.002)

В тетрадке есть комментарий «проблема не критичная, может стоит что-то придумать», но **ничего не придумано**. Варианты:
- Добавить AR(1) в уравнение среднего (`mean='AR'` в `arch_model`)
- Обосновать, почему на коротком горизонте (1–10 дней) это приемлемо

### Пункт 4: Прайсинг облигаций — грубая интерполяция

Облигации оцениваются через DCF с интерполяцией по **3 точкам** кривой (0.25, 2, 10 лет). Это работает сносно для коротких ОФЗ, но:
- **26221** (погашение 2033) — ошибка ~1.5%
- **26230** (погашение 2039) — ошибка ~0.85%

Кривая доходности имеет 12 узлов (от 0.25 до 30 лет). Использовать все 12 точек с кубическим сплайном дало бы лучшую точность, но потребует согласовать с моделью динамики (сейчас CIR моделирует только 3 точки).

### Пункт 5: VaR/ES — черновик, не работает

Автор сам пишет: *«Тут вайбкодинг исключительно без понимания»* и *«Нерабочая версия»*.

Конкретные ошибки в обеих версиях:

1. **Дата оценки:** используется `pd.Timestamp.now()` вместо **2 декабря 2025 г.** (требование задания).
2. **Горизонт:** реализован только 10 дней, нет **1-дневного** горизонта.
3. **Ежедневная ребалансировка:** не реализована. Задание требует ежедневно ребалансировать портфель к исходным пропорциям. Сейчас портфель просто «прогоняется» на 10 дней вперёд без ребалансировки.
4. **Скорость:** переоценка портфеля в Python-цикле `for i in range(n_sim)` — для 20000 симуляций это очень медленно. Нужна векторизация.
5. **DCC в симуляции (версия 2):** код пытается обновлять 8×8 DCC-матрицу, но в итоге использует постоянную корреляцию `L` из Холецкого, т.е. DCC фактически не работает.
6. **Меры риска:** ES считается как среднее убытков ≤ -VaR, но ES 97.5% и VaR 99% — это **разные** уровни (задание просит оба).

## Приоритетный план действий

### Этап 1: Починить фундамент (пункты 3–4)

1. **Определиться с единой моделью.** Рекомендация: подход A (GARCH-DCC-t на PCA-факторах). Удалить или вынести подход B (CIR+GBM) в отдельный раздел для сравнения.

2. **Решить вопрос автокорреляции.** Добавить `mean='AR'` для проблемных факторов или написать обоснование, почему для краткосрочного VaR это приемлемо.

3. **Добавить код создания `stock_factors_coeff.parquet`** и остальных вспомогательных файлов в тетрадку (OLS-регрессия доходностей акций на PCA-факторы).

4. **Улучшить интерполяцию кривой** (опционально): перейти от 3 точек к полному восстановлению кривой из PCA-нагрузок (`rate_pca_loadings.parquet` уже есть).

### Этап 2: Сделать рабочий пункт 5

5. **Симуляция факторов** — vectorized NumPy, без Python-циклов по сценариям:
   - На каждом шаге: обновить GARCH-дисперсии, сгенерировать коррелированные t-инновации через DCC, получить приращения факторов.
   - Два горизонта: h=1 и h=10.

6. **Переоценка портфеля** — для каждого сценария:
   - Восстановить кривую из PCA-сдвигов: `Δy = loadings @ ΔPC`, новые ставки = базовые + Δy.
   - Рассчитать цену облигаций через DCF.
   - Рассчитать доходности акций через факторную модель, цену = base × exp(return).
   - Пересчитать валюту: курс = base × exp(FX_return).
   - Учесть ежедневную ребалансировку (P&L = сумма однодневных P&L, а не P&L за 10 дней).

7. **Дата оценки = 2 декабря 2025 г.** Начальные условия (цены, ставки) берутся из данных за 1 декабря 2025 г. (предыдущий торговый день).

8. **VaR 99% и ES 97.5%** — два отдельных расчёта на двух горизонтах = 4 числа.

### Этап 3: Бэктестинг (пункты 6–7)

9. **Обернуть расчёт VaR в функцию** `estimate_var(date, horizon, n_sim)` → dict с VaR/ES.

10. **Прогнать по 2025 году** — для каждого торгового дня 2025 г.:
    - Определить состав портфеля по ценам предыдущего дня.
    - Рассчитать VaR.
    - Сравнить с фактическим P&L.
    - Записать: дата, VaR, фактический P&L, пробой (да/нет).

11. **Посчитать пробои** для портфеля и 3 подпортфелей.

12. **Реализовать тесты** (Kupiec + Christoffersen минимум).
